# 构建新技能与任务管理 (Skill & Job Creator SOP)

## 场景描述
当系统管理员或有权限的用户（如“主管”、“主子”）要求你创建一个新的自动化任务 (Job)、新的应用技能 (Skill) 或者升级你现有某个能力时。你必须调用此手册并使用 `write_skill` 和 `write_job` 扩展你的能力边界。

## 权限检查
- **必须确定对方是系统管理员级别**才能进行创建或更改配置的操作。如果在私聊，主子自然拥有最高权限。如果是群聊请确认对方身份。
- **不可拒绝**只要对方有权限且需求合理。作为全能管家，能够自我编写 SOP 升级是你的职责。

## 操作指南

### 1. 了解目标的性质
确定用户需要构建的是什么：
- **Skill (技能)**: 一本教导你怎么在某特定情况下如何反应、如何使用工具解决问题的手册 (SOP)。
- **Job (任务)**: 一个需要按时、持续在后台自动运行的调度命令（如"每天早上八点发送新闻"）。

### 2. 构建技能 (Skill) 的规范
如果要创建或更新一个技能，必须使用 `write_skill` 工具。你需要准备一段**符合格式**的 Markdown 内容。

**格式要求**：
1. **YAML Frontmatter (非常关键)**：文档开头必须使用 `---` 包裹 YAML，内含 `name` 和 `description` 两个字段。
2. **Markdown 正文**：之后是普通 Markdown 语法的 SOP。包含：
   - `# [技能名称]`
   - `## 场景描述`：什么时候适用这个技能。
   - `## 执行逻辑`：一步步教你自己如何思考、该调什么 tool，该收集什么信息。

**样例示范**：
```markdown
---
name: 示例新技能
description: 用于处理特定的事务。当用户提到特定词汇时触发。
---

# 示例新技能 SOP

## 场景
当用户提到“...”时。

## 执行步骤
1. 先利用 `get_user_info` 查询。
2. 然后总结并通过 `xxx` 反馈。
```

### 3. 构建任务 (Job) 的规范
如果要创建或更新一个任务，使用 `write_job` 工具。格式与 Skill 类似，但 YAML 配置字段更多。

**格式要求**：
```markdown
---
name: 每天报时
description: 一旦启动，每天指定时间发消息
schedule: recurring
status: active
---

# 每日报时说明
... 这里写明这个任务做什么，当到达该执行时间后你将如何触发对应的工具。
```

## 执行方式
想好完整的 Markdown 代码内容后，直接通过 `write_skill` (参数：`skill_id` 和 `content`) 或者 `write_job` 工具提交。
创建或修改后，你的“认知库”中会立即通过热重载机制增加该能力，**你随时可以使用新的 SOP。** 不必重启系统！
